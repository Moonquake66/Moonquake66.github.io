linux基础命令及shell使用

1.文件和目录操作命令

1.1 ls命令

用于列出目录的内容，包括文件和子目录。

1.2 cd命令（用于切换当前工作目录）

1.3 pwd命令

打印当前工作目录的绝对路径。

1.4 mkdir命令

用于创建新的目录。

1.5 rmdir命令

用于删除空目录。

1.6 rm命令

用于删除文件或目录。

1.7 CP命令

用于复制文件和目录。

1.8 mv命令

用于移动或重命名文件和目录。

2.文件查看和编辑命令

2.1 cat命令

用于查看文件内容、连接文件并输出到标准输出。

2.2 less命令

用于分页查看文件内容，比cat命令更灵活，适合查看大型文件。

2.3 head命令

用于查看文件的开头部分内容。

2.4 tail命令

用于查看文件的结尾部分内容。

3.系统信息和进程管理命令

3.1 uname命令

用于打印系统信息。

-a：打印所有系统信息，包括内核名称、主机名、内核版本、硬件平台等。

3.2 top命令

动态显示系统中各个进程的资源占用情况，如 CPU 使用率、内存使用率等。

按下 “q” 键退出top命令。

3.3 ps命令

用于查看当前系统中的进程状态。

-ef：显示所有进程的详细信息，包括进程 ID（PID）、父进程 ID（PPID）、用户、CPU 使用率等。

3.4 kill命令

用于终止进程。

灰度发布

什么是灰度发布

金丝雀发布有名灰度发布。起源于17世纪，英国矿井工人发现，金丝雀对瓦斯这种气体十分敏感。空气中哪怕有极其微量的瓦斯，金丝雀也会停止歌唱；而当瓦斯含量超过一定限度时，虽然人类毫无察觉，金丝雀却早已毒发身亡。当时在采矿设备相对简陋的条件下，工人们每次下井都会带上一只金丝雀作为“瓦斯检测指标”，以便在危险状况下紧急撤离。

灰度发布流程

  • 在金丝雀发布开始后，先启动一个新版本应用，但是并不直接将流量切过来，而是测试人员对新版本进行线上测试，启动的这个新版本应用，就是我们的金丝雀。如果没有问题，那么可以将少量的用户流量导入到新版本上，然后再对新版本做运行状态观察，收集各种运行时数据，如果此时对新旧版本做各种数据对比，就是所谓的A/B测试。

​

  • 当确认新版本运行良好后，再逐步将更多的流量导入到新版本上，在此期间，还可以不断地调整新旧两个版本的运行的服务器副本数量，以使得新版本能够承受越来越大的流量压力。直到将100%的流量都切换到新版本上，最后关闭剩下的老版本服务，完成金丝雀发布。

  • 如果在金丝雀发布过程中（灰度期）发现了新版本有问题，就应该立即将流量切回老版本上，这样，就会将负面影响控制在最小范围内。

总之，金丝雀发布指的是在生产环境中分阶段逐步更新后端应用的版本（需要具备流量控制能力），在小范围验证符合预期之后，再推广至整个生产环境。

OpenResty介绍

简单的说uaOpenResty=nginx+LuaJIT ,它集成了我们的 Nginx核心的增强版,我们的LuaJIT增强版，许多精心编写的Lua 库，许多高质量的第三方Nginx模块，以及 他们的大多数外部依赖关系。

就是在原本普通版本的nginx的基础上将lua脚本与之结合。Web 开发人员可以使用 Lua 脚本语言调动 Nginx 支持的各种 C 以及 Lua 模块,更主要的是在性能方面，OpenResty可以 快速构造出足以胜任 10K 以上并发连接响应的超高性能 Web 应用系统。

OpenResty® 的目标是让你的Web服务直接跑在 Nginx 服务内部，充分利用 Nginx 的非阻塞 I/O 模型，不仅仅对 HTTP 客户端请求,甚至于对远程后端诸如 MySQL、PostgreSQL、Memcached 以及 Redis 等都进行一致的高性能响应。
